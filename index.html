Homepage Section Order — Mobile Priority
Scroll order on phone = what matters most
MVP philosophy: Community first — events, social, news. Get those three right and people have a reason to come back daily. Stores and commerce follow once there's an audience.
1️⃣
Upcoming Events — Eventbrite API. Date-sorted, soonest first. Filter pills for This Week / Weekend / Tastings / Classes / Free.

#nycwine Social Feeds — Instagram (Elfsight), X timeline, Facebook page, Pinterest. Placeholder shown until each widget is configured.

4️⃣
NYC Wine Stores — Simple borough-filtered list. Name, neighborhood, tags, link to their website. Data from our store database. No ordering — all offsite.

Neighborhood filter → store results

The user selects their borough, then their neighborhood. The site instantly filters the wine store database to show the closest stores — with name, address, phone number, and a "Visit Website" link. All ordering happens on the store's own site.

4
Wine Store of the Week
Rotating spotlight — fully automated

Each week, a different wine store from the database is featured with a short editorial description, their address, phone, and a link to their website. This gives every store in the database a chance to be highlighted, and gives users a reason to check back.

How the Rotation Works
The system cycles through all "registered" stores in the Supabase database in a set order. No manual work required after initial setup.
Every Monday
Featured store automatically advances to the next in the queue
Queue logic
Round-robin rotation — each store gets equal time before repeating
New stores
Added to the end of the queue when you add them to the database
Premium option
Featured stores could eventually pay for placement (future revenue stream)
🔌
NYCWine Database · Supabase No external integration needed — purely our data. Store info (name, description, photo, phone, website) managed in the database.

How it works: Next.js fetches each RSS feed server-side every hour. Results are merged, deduplicated, and sorted newest-first. Source tabs filter client-side — no additional API calls needed.
💰
Cost: Free. All RSS feeds are free. Reddit API free tier allows 100 calls/minute — more than enough for hourly polling.

Filtering note: NYT and NY Post RSS feeds are full-section feeds. We'll keyword-filter client-side for "wine" terms so non-wine stories don't appear.

Google Ads — Action Required
Clarify account type before integrating

Finding from research: Google AdSense (the standard "show ads on my website" product) has restrictions that can limit wine/alcohol content sites. Google Ad Manager (the professional publisher tier) handles wine content fine with proper settings.

❓
Action needed: Check whether the existing Google account set up for NYCWine is:
(A) Google Ads — for running search/display ads to bring people to the site
(B) Google AdSense — for showing ads on the site and earning revenue
(C) Google Ad Manager — professional publisher ad platform

If it's (A), we still need to set up (B) or (C) to show ads on the site. If it's (B), we need to verify wine content is approved. If (C), we're in good shape.
💡
Backup options: If AdSense blocks wine content, we can use direct ad sales (sell banner space to wine brands/stores directly, no platform needed) or look at Mediavine/AdThrive for wine-friendly networks once the site has traffic.
